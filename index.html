说说云服务的一些事

发表于 2017-03-25 | 分类于 work

看了下，都快好几个月都没更新了，昨天突发奇想，看了下PR值，居然也有2了！但愿我后面的日子会好过点。

没什么好说的，随便提出了几个关键词，跟大家分享下。

1. Linux

我最近的工作都是跟linux打交到，学了很多东西，跟自学的时候差别很大，因为后面有人用枪顶着。

Shell：很好很强大的一个工具，我不想把他定义为语言，但只是工具，处理手中杂事的一种高效的工具，如果有一件事情你需要做两边以上，那么写一个shell来完成把～～

Ubuntu：上周开始试着放弃windows，改用linux，对我个人而言确实方便了很多，加上Vbox来处理那些linux无法完成的任务，其余的时间就都跟它度过了。本来在办公室安装的是10.10，但回家体验了下11.04以后，果断换成了Unity界面，符合我的风格。

MacOS：周末两天都耗在这个上面了，好看，不知道好不好用，把驱动那些东西还没看懂，目前处于mac无法联网的状态，黑苹果继续“黑”着它的眼睛（因为无法上网呀）。

2. 云服务

GAE：最早接触的云服，做的不错，毕竟是google的东西嘛，那技术让人望其项背呀！因为目前只支持python和java，又被GFW挡着，我也只有看看了～～

SAE：看这就象山寨，其实他也是山寨，连名字都差不多，目前只提供php的服务，技术水平还不敢发表评论，但你的应用要是建在这个上面，那后面的日子估计不好过，玩玩还可以了。。

Amazon：强大的亚马逊，直接提供给你主机，霸气威武不解释，应该是可以用他来看墙外的世界的，今天下午信用卡被扣了款，还没开通成功，郁闷死我了～～

DotCloud：很有意思的一个云服务，我以为后面是什么组织呢，建了一个应用以后，在ubuntu下ping了下域名，果断把我雷到了，下面是真相

看的清吗？\^_\^，他用的居然是“AWS”，小子建在服务之上的服务，高，实在是高！

其实这未免也不是一种好的想法，把一种服务包装一下，变成另外一个服务，然后用前面的服务来支撑后者，顺便还有点结余～～恩，不错！

好了，不扯了，该下班了～～

说说我之前的工作（一）

发表于 2017-03-25 | 分类于 work

很久没写东西了，以后的工作也会多多少少的跟文字沾上一点边。原因之一是锻炼自己的写作能力，第二也想重新反思下工作这几年的沉淀，所以有了这篇文章。能写多久，会写多少，看缘分了。

网络工程师@某网络设备厂商

1. 跟我一起入职的网络工程师现在都在干嘛？

毕业已经看起来是很久远的事情了。已经不太记得为什么工作的第一份工作选择网络，事实上在工作一年半左右的时间，大家就纷纷转岗了，大部分去做了网络设备的测试或是售前咨询的岗位。

说实话，这是一个门槛不是很高的行业，发展前景也不乐观的一个职业，如果你是毕业生看到这篇文章，请谨慎选择。

2. 如何做好这份工作？

想要做好这份工作，我是最没有发言权了，因为我本身就没有做的太好。一个是公司对你工作岗位定位，另外一个是个人兴趣导向。使得我自己在这个方向上没做出什么成绩，不过，我看到身边的一些做的好的同事，他们一些特有的素质。

  1. 专业性，理论知识需要打牢固，如7层网络模型各层的协议你了解了多少
  2. 实践，有了理论就需要实践，没有现成的机器需要装模拟器，装虚拟机
  3. 认证，各种思科认证CCNA/CCNP/CCIE，华为认证你到底能到哪一层
  4. 态度，态度决定一切，在任何时候都会适用

3. 这一年，我都做了啥？

公司不是学校，不会给你钱还让你学东西，作为职场的你，一定要清楚，你为公司创造了多少价值？

  1. Helpdesk，负责了公司电脑和网络的维护工作
  2. 新园区的建设，网络规划和实施，其中还有部分布线
  3. 学会了一个叫Routeros的技能，现在已经在两所学校实施了这一成果
  4. 新园区的自动化配置建设（现在看来就是企业内部的一个cmdb）

最后一个做的事情现在看来最有意义，让接下来的维护的一个同学转行做了php开发，也有幸让我进入了互联网行业。

4. 这份工作让我学到了什么？

一个懵懂的学生，初入职场，没有经历过什么尔虞我诈，没有勾心斗角，一心只为学技术。

  1. 收获了到现在还偶尔还会出来吃吃东西的好朋友
  2. 收获了到现在还会用到的网络技巧
  3. 收获了很好的自学的习惯，凡事都要自己去摸索
  4. 收获了工作的态度，凡事答应了就要做到，做到就要做好
  5. 还收获了在深圳活下去的第一笔收入

回顾2011，展望2012！

发表于 2017-03-25 | 分类于 life

起初，我把这个博客的定位在技术笔记上，后来发现这个后台实在是太难用了（相对与mediawiki来说），慢慢的，写的就少了。更多的时候，把笔记都写在私人的evernote上面了，时间管理的东西就用doit.im了。博客也慢慢的变成了一片荒草地，没办法，现在感觉时间太少了，以至于更新一篇都如此的难！下面就来记流水帐了…

2011

在这一年，对于我个人来说是不平凡的一年，几个关键字可以归纳一下。离职&入职、无方向=>互联网、RouterOS=>Linux、靠谱的公司&靠谱的老板&给力的导师。

离职&入职

5月份上半月，在拿到剩下的一半年终奖以后，我选择了离职。离开了我职业生涯的第一个公司xx”一家看似技术型的公司”。可能我在这里的职位“网络工程师”太尴尬了，所以在被闲置了大约半年的时间以后，我选择了离开，因为实在是找不到继续呆下去的理由了。从心底里讲，我还是挺感激它的，至少它给了我在深圳生活下去最基本的物质保障，给我上了职业生涯的第一课，在这里认识了一大群跟我一样青涩的大学毕业生们，虽然现在还呆在这里的已经不足一半了。直到现在，我们还会在群里侃侃自己的生活，有时间找几个朋友小聚，非常难忘的职业生涯的起点。

5月份下半月，上天让我非常有缘分的加入了xx，这个成立刚过半年的创业公司，从5月份到现在半年多的时间，我认为是我自己在毕业以后成长最快的半年，学的知识也是最多的半年，收获也是最多的半年，所以我非常感谢xx，这个温馨而美好的大家庭。

无方向=>互联网

毕业以后，才明白现实与理想的差距实在是太大了。有人说毕业后的第一份工作非常重要，而我不这样认为，大多数的同学都是在自己不知道自己喜欢什么的情况下选择了一份工作，或者说是被迫选择了一份工作（比如说我），当工作一段时间以后，也许有些同学还是不知道自己要什么，索性就把这个工作当作自己的职业一直继续下去了,当明白自己的想要的东西以后，又舍不得自己毕业后所经历的。于是我就武断的下了一个结论：毕业以后换的第一份工作是最重要的。有些同学换工作仅仅为了薪水 的增加，如果是这样子的话，离下一次离职也不远了。我觉得大多数同学换了一份工作以后都会把它当作毕生的事业来做，所以换工作一定要谨慎。

很小的时候，我比同龄的孩子更早的接触了互联网，选择互联网算是小时候好奇的一种回归吧。

RouterOS=>Linux

以前我的blog大部分是关于网络部分的，最典型的是routeros的东西写的比较多，而且大部分是通过讲原理的方式来让自己对网络的路由交换原理理解的更加透彻，而且我很不喜欢将一件很简单的事情重复的去做很多遍，比如在交换机上配置vlan这种低脑力劳动，高体力劳动，我更愿意让计算机去帮我完成，所以我喜欢上了脚本，大家可以看到后面routeros的很多东西我都是写的脚本在上面。慢慢的，我发现网络只能解决一些基础问题，按照TCP/IP的分层理论的话，它解决不了第7层的问题，给你网络设备，你只在那边ping来ping去是没有任何意义的，所以我发现了网络设备一定要跟网络服务器结合在一起用，才是现代互联网的归属。而我正在转型，从关注基础服务多一点，到关注应用服务多一点，老实地说，玩linux的成就感比玩网络的成就感要大很多。也许这也是我的导师所说的从偏硬在向偏软转换吧。

靠谱的公司&靠谱的老板&给力的导师

我很幸运在加入xx以后，逐渐感受到这是一个靠谱的公司，认认真真在做事情的一家公司，老板基本上是每天最晚回家的那一个人，你还能不说这是一个靠谱的老板么？在我入职还不到半年的时候拿到公司的期权，而且在今年的年会上，老板还说要给这些期权分红，你还能hold住吗？最重要的是，到了公司以后，有一个非常给力的导师，无论在技术上还是解决问题的思路、态度以及心态上，都给了我很多的帮助，这将是我在今后的人生道路上的宝贵财富。我只想轻轻的对公司，对老板，对导师说：“谢谢！我不会辜负你们对我的期望！”

2012

无论2012是否是世界末日，我是否能拿到船票，都希望自己能完成下面几件事情：

  1. 用足够多的时间去关心家人，关心朋友，关心亲爱的她；
  2. 努力的工作，尽职尽责的完成老板交给我的每一项任务；
  3. 去祖国的大好江山（非湖北、广东）的某一个地方溜一圈；
  4. 未完待续……

（ps:因为某种原因，我略去了公司名称，请见谅！）
